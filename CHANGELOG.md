# 更新日志

## v1.0.1

此版本集中发布 `v1.0.0` 之后完成的管理能力、安全性和可靠性改进。

### 管理与路由

- 新增 Minecraft 玩家身份绑定，并扩展玩家、世界、白名单、封禁、游戏规则、标题和自定义命令等管理操作
- 完善多服务器、多QQ群路由以及消息投递结果，区分全部成功、部分成功和全部失败
- 自定义命令会校验服务器、命令名、参数数量和控制字符，避免模型幻觉生成非法命令

### 权限与安全

- 保持 LLM 默认 `READONLY`；所有写工具统一要求管理员身份和 FULL 模式
- 管理员可随时紧急恢复 READONLY，降权操作不受普通限流影响，开启 FULL 仍需精确确认
- 结构化 RCON 操作增加目标、枚举、数值范围和控制字符校验，并串行执行共享连接上的命令
- 完善 `.gitignore`，避免本地配置、凭据、证书、运行时绑定和发布产物被误提交

### MCSManager

- 支持默认校验、自定义 CA、首次信任证书和显式关闭校验等按面板 TLS 策略；首次信任失败时不再静默降级
- API 或 TLS 请求失败会返回明确错误，不再把故障误报为空面板、空实例、空目录或空文件
- 多面板部分故障时，具体实例操作要求显式指定面板，避免目标歧义
- 加强实例、文件、日志和控制台响应校验，并从 HTTP 日志中脱敏 `apikey`

### 稳定性与兼容性

- WebSocket 临时断线会持续使用有上限的退避重连，并改进关闭超时和死亡事件载荷兼容
- QQ 和 RCON 消息统一使用安全 JSON 序列化，MCSManager 命令已发送但日志读取失败时会明确报告部分成功
- 依赖版本限制在已验证的兼容范围；GitHub Actions 覆盖 Python 3.10/3.12、AstrBot 4.10.4/4.26.7 和一次性 Minecraft 1.21.1 RCON

## v1.0.0

首个正式公开版本。此版本整合此前开发阶段的全部功能与修复，不再将内部开发里程碑拆分为多个公开版本。

### 多服务器统一管理

- 支持配置多台命名 Minecraft 服务器，每台服务器拥有独立的 RCON、WebSocket、消息同步和危险命令策略
- 管理目标按显式服务器ID、用户当前选择、默认服务器的顺序解析，与QQ群消息路由相互独立
- 支持查询全部服务器的连接状态和在线玩家，并提供命令与 LLM 工具两种操作方式

### 多QQ群消息路由

- 每台服务器卡片可直接填写多个 `qq_group_ids`；同一群号也可填写到多台服务器，实现多对多路由
- 支持 MC→QQ、QQ→MC、玩家事件和可选的 LLM 回复转发
- WebUI配置与 `/mc bind` 创建的运行时绑定合并生效，并明确标注绑定来源
- 自动迁移早期分组式绑定配置，保存失败时完整回滚

### 权限与安全

- LLM 默认使用 `READONLY` 模式；写操作必须同时满足 FULL 模式和 `admin_ids` 管理员白名单
- 模型不能自行提升权限；FULL 模式只能由管理员使用精确确认指令开启
- RCON 与 MCSManager 统一拦截危险命令变体，默认禁止 `stop`、`reload` 等操作
- 操作日志不记录完整密码、命令响应或原始 WebSocket 载荷

### MCSManager

- 支持多个面板和实例的状态查询、启动、停止、重启、命令执行与日志读取
- 支持只读目录列表和文件读取，拒绝路径穿越和 NUL 字符，单次内容限制为 12000 字符
- 多面板查询并发执行，并兼容当前官方分页、状态和文件类型返回格式
- 文件写入、删除、移动、上传和下载尚未开放

### 质量与兼容性

- 遵循 AstrBot 当前插件结构、命令组、消息链和标准数据目录规范
- 支持 AstrBot 4.10.4 及以上版本
- GitHub Actions 覆盖 Python 3.10/3.12、AstrBot 4.10.4/4.26.7 加载测试和一次性 Minecraft RCON 集成测试
