# 更新日志

## v1.0.0

首个正式公开版本。此版本整合此前开发阶段的全部功能与修复，不再将内部开发里程碑拆分为多个公开版本。

### 多服务器统一管理

- 支持配置多台命名 Minecraft 服务器，每台服务器拥有独立的 RCON、WebSocket、消息同步和危险命令策略
- 管理目标按显式服务器ID、用户当前选择、默认服务器的顺序解析，与QQ群消息路由相互独立
- 支持查询全部服务器的连接状态和在线玩家，并提供命令与 LLM 工具两种操作方式

### 多QQ群消息路由

- 每台服务器卡片可直接填写多个 `qq_group_ids`；同一群号也可填写到多台服务器，实现多对多路由
- 支持 MC→QQ、QQ→MC、玩家事件和可选的 LLM 回复转发
- WebUI配置与 `/mc bind` 创建的运行时绑定合并生效，并明确标注绑定来源
- 自动迁移早期分组式绑定配置，保存失败时完整回滚

### 权限与安全

- LLM 默认使用 `READONLY` 模式；写操作必须同时满足 FULL 模式和 `admin_ids` 管理员白名单
- 模型不能自行提升权限；FULL 模式只能由管理员使用精确确认指令开启
- RCON 与 MCSManager 统一拦截危险命令变体，默认禁止 `stop`、`reload` 等操作
- 操作日志不记录完整密码、命令响应或原始 WebSocket 载荷

### MCSManager

- 支持多个面板和实例的状态查询、启动、停止、重启、命令执行与日志读取
- 支持只读目录列表和文件读取，拒绝路径穿越和 NUL 字符，单次内容限制为 12000 字符
- 多面板查询并发执行，并兼容当前官方分页、状态和文件类型返回格式
- 文件写入、删除、移动、上传和下载尚未开放

### 质量与兼容性

- 遵循 AstrBot 当前插件结构、命令组、消息链和标准数据目录规范
- 支持 AstrBot 4.10.4 及以上版本
- GitHub Actions 覆盖 Python 3.10/3.12、AstrBot 4.10.4/4.26.7 加载测试和一次性 Minecraft RCON 集成测试
